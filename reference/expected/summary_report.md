## 7. Decision notes

- Dependency cache reduced the dependency preparation phase by -16.1 % and the measured total time by -3.9 %.
- The main bottleneck in the optimized backend cache scenario is Tests (median 36.203 s).
- Docker build cache reduced the Docker build phase by -56.8 % and the measured total time by -43.4 %.
- The main bottleneck in the Docker baseline scenario is Docker build (median 110.000 s).
- On the original test suite, timing-based sharding did not reduce the critical path. The median critical path changed by 0.9 %, which indicates that the original workload had limited room for improvement.
- On the original test suite, the imbalance ratio changed by 0.2 %.
- On the extended test suite, timing-based sharding reduced the critical path by -6.9 %, which indicates a measurable improvement of the parallel test execution path.
- On the extended test suite, the imbalance ratio changed by -7.8 %, which is used as an indicator of shard balance.
- On the extended test suite, the difference between the slowest and fastest shard changed by -45.5 %.
- The sharding results indicate that timing-based distribution is more useful when the test suite has sufficient granularity. This supports interpreting test-suite structure as a key condition for effective parallelization.
